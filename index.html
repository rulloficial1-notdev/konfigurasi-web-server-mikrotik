LAB Web Server MikroTik

Halaman ini berisi dokumentasi lengkap konfigurasi router MikroTik untuk kebutuhan routing, DHCP, NAT, wireless access point, dan web server lokal. Susunan isi dibuat agar bisa langsung dipakai sebagai bahan laporan, presentasi, maupun panduan praktik step by step.

Informasi Cepat

  • Platform: MikroTik RouterOS via Winbox
  • Mode Lab: Dapat berjalan lokal tanpa internet
  • Layanan: DHCP, WiFi AP, HTTP port 80 dan 8080
  • Dokumen diperbarui: 26 Maret 2026

Ringkasan Dokumentasi

Sebelum masuk ke tahap konfigurasi, bagian ini menjelaskan tujuan praktikum, perangkat yang diperlukan, dan gambaran topologi agar proses setup lebih mudah diikuti.

Tujuan Praktikum

  • Mengonfigurasi router MikroTik sebagai pusat koneksi beberapa jaringan.
  • Menyediakan alamat IP otomatis untuk server LAN, client LAN, dan WiFi.
  • Mengaktifkan layanan web server lokal agar file index.html dapat diakses melalui browser.

Kebutuhan

  • 1 router MikroTik yang dapat diakses melalui Winbox.
  • Perangkat client untuk pengujian melalui LAN atau WiFi.
  • File web statis index.html yang diunggah ke menu Files.

Catatan Penting

  • Langkah default route dan NAT hanya dibutuhkan jika lab terhubung ke internet.
  • Jika hanya untuk akses lokal, web server tetap bisa berjalan tanpa koneksi internet.
  • Gunakan satu skenario IP secara konsisten agar pengujian tidak rancu.
ether1 - Jalur ISP / Internet (opsional) 192.168.1.2/24
ether2 - Server LAN 192.168.100.1/24
ether3 - Client LAN 192.168.200.1/24
wlan1 - WiFi Access Point 10.10.1.1/24

Langkah Konfigurasi

Urutan berikut sudah disusun dari tahap awal sampai pengujian akhir. Format ini cocok digunakan sebagai isi utama laporan dokumentasi praktikum.

Reset router ke kondisi bersih

Buka menu System > Reset Configuration, lalu aktifkan opsi No Default Configuration. Langkah ini memastikan router tidak membawa aturan lama yang dapat mengganggu konfigurasi lab.

Atur IP address pada tiap interface

Masuk ke menu IP > Addresses lalu tambahkan alamat IP sesuai fungsi jaringan yang digunakan pada lab.

ether1 = 192.168.1.2/24 (ISP / Internet) ether2 = 192.168.100.1/24 (Server LAN) ether3 = 192.168.200.1/24 (Client LAN) wlan1 = 10.10.1.1/24 (WiFi)

Tambahkan default route jika ada koneksi internet

Langkah ini hanya diperlukan bila router akan mengakses jaringan luar. Jika lab murni lokal, bagian ini dapat dilewati tanpa memengaruhi web server internal.

Menu: IP > Routes > Add Dst. Address : 0.0.0.0/0 Gateway : 192.168.1.1

Konfigurasi DNS

Buka menu IP > DNS lalu isi server DNS dan aktifkan Allow Remote Requests agar client dapat menggunakan router sebagai resolver.

Primary DNS : 8.8.8.8 Secondary DNS : 8.8.4.4 Allow Remote Requests : enabled

Buat DHCP server untuk setiap segmen

Jalankan wizard pada menu IP > DHCP Server untuk membagikan IP secara otomatis kepada perangkat yang terhubung ke masing-masing jaringan.

ether2 -> 192.168.100.10 - 192.168.100.100 ether3 -> 192.168.200.10 - 192.168.200.100 wlan1 -> 10.10.1.10 - 10.10.1.100

Aktifkan wireless access point

Masuk ke menu Wireless > wlan1, ubah mode menjadi ap bridge, lalu tentukan SSID dan keamanan WiFi.

Mode : ap bridge SSID : LAB-ARULL Security : WPA2-PSK Password : 12345678

Tambahkan NAT bila internet dibagikan ke client

NAT hanya dipakai saat client perlu mengakses internet melalui interface menuju ISP. Untuk akses lokal ke web server MikroTik, aturan ini tidak wajib.

Menu : IP > Firewall > NAT Chain : srcnat Out. Interface: ether1 Action : masquerade

Aktifkan layanan web server

Buka menu IP > Services lalu aktifkan service www pada port 80. Jika dibutuhkan, duplikasi service untuk port 8080.

www -> enabled on port 80 www-8080 -> enabled on port 8080

Upload file halaman web

Masuk ke menu Files lalu unggah file index.html. Setelah file berada di router, halaman dapat diakses melalui browser client yang berada dalam jaringan.

File yang diunggah: index.html

Lakukan pengujian akses dari WiFi dan LAN

Buka browser pada perangkat client lalu uji akses menggunakan alamat IP sesuai segmen yang dipakai. Jika service port 8080 aktif, lakukan pengujian tambahan pada port tersebut.

Dari WiFi: http://10.10.1.1 http://10.10.1.1:8080 Dari Server LAN: http://192.168.100.1 Dari Client LAN: http://192.168.200.1

Penjelasan routing pada skenario ini

Karena semua subnet berada di dalam satu perangkat router, proses routing antarjaringan sudah berlangsung otomatis selama tiap interface memiliki IP yang benar dan tidak ada firewall yang memblokir lalu lintas internal.

Dokumentasikan hasil akhir dan troubleshooting

Simpan bukti konfigurasi, hasil ping, serta tangkapan layar tampilan web server sebagai bagian dari laporan. Catat juga masalah yang muncul dan tindakan koreksinya.

Poin cek: - DHCP server berjalan normal - Client mendapat IP sesuai pool - Service www aktif - Port 80 / 8080 tidak diblokir firewall - File index.html berhasil terbaca oleh router

Checklist Pengujian

Bagian ini membantu memastikan dokumentasi tidak hanya berisi konfigurasi, tetapi juga hasil verifikasi bahwa seluruh layanan berjalan sesuai tujuan.

Uji Konektivitas

Pastikan client memperoleh IP otomatis, dapat ping ke gateway, dan dapat menjangkau router sesuai segmen jaringan.

Uji Layanan Web

Buka akses HTTP dari WiFi dan LAN. Verifikasi bahwa isi halaman tampil tanpa error pada port 80 maupun 8080 bila diaktifkan.

Uji Internet Opsional

Jika ether1 terhubung ke ISP, cek DNS resolution dan akses internet dari client untuk memastikan route dan NAT sudah benar.

Catatan Tambahan

Kamu bisa memakai bagian ini sebagai penutup laporan untuk menjelaskan hasil praktikum dan hal-hal yang perlu diperhatikan saat implementasi ulang.

  • Kesimpulan: MikroTik dapat digunakan sebagai pusat routing, DHCP, WiFi, dan web server lokal dalam satu perangkat.
  • Keamanan: Ganti password WiFi dan batasi service yang tidak digunakan jika lab akan dipakai di lingkungan nyata.
  • Dokumentasi laporan: Tambahkan screenshot Winbox, hasil pengujian browser, dan konfigurasi interface sebagai lampiran.
